Agent Interaction Flow

How users interact with R&D IT AI Agents through the Agent Mesh Framework

1
🖥️

User Request

User submits a task via the Chat UI interface (e.g., "Generate tests for Module 3 submission")

Client → Route 53 → ALB
→
2
🔐

Authentication

Request authenticated via Cognito/Entra ID SSO. PEP validates token and enforces policies.

ALB → Cognito → PEP
→
3
⚡

Session & Cache

User Gateway validates session, stores context in ElastiCache Redis for fast retrieval.

Frontend-ECS → ElastiCache
→
4
📬

Task Queued

Task submitted to SQS queue for async processing. Ensures reliable delivery and scaling.

Frontend-ECS → SQS
→
5
📚

Agent Discovery

Orchestrator queries Agent Registry, selects appropriate agent (e.g., Test Intelligence).

Orchestrator → Agent Registry
6
📜

Policy Check

PDP validates permissions against GxP compliance rules. Ensures proper data access controls.

Orchestrator → PDP
→
7
🔀

LangGraph Execution

Agent loop runs via LangGraph state machine. Checkpoints saved to DynamoDB for recovery.

Agent Loop → LangGraph → DynamoDB
→
8
🔌

Tool Invocation

MCP Client calls external tools (3DP, BIKG, GitHub, Confluence) via MCP Server protocol.

MCP Client → MCP Server → Tools
→
9
🤖

LLM + Guardrails

Agent calls Bedrock (Claude 3.5). Lambda AI Judge validates. Guardrails ensure safe output.

Agent → Lambda → Bedrock + Guardrails
→
10
📤

RAG & Response

Context retrieved via RAG from OpenSearch. Results stored in S3. Response returned to user.

OpenSearch → S3 → User

Key AWS Services

ECS Fargate Bedrock (Claude) Bedrock Guardrails Lambda (AI Judge) OpenSearch ElastiCache Keyspaces DynamoDB S3 SQS Cognito Route 53 ALB Secrets Manager

Agent Mesh Components

  • Agent Registry: Discovers and routes to registered agents
  • Tool Registry: Catalogs available MCP tools and services
  • PDP (Policy Decision Point): Enforces GxP compliance rules
  • PEP (Policy Enforcement Point): Auth, rate limits, schema validation
  • MCP Server/Client: Standard protocol for agent-tool communication
  • LangGraph: State machine orchestration for agent execution
  • Instrumentation: OpenTelemetry, Prometheus, Grafana observability

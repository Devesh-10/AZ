Metrics Explained

Test Intelligence Agent — Understanding Each Dashboard Metric

1 Key Performance Indicators (KPIs)

Test Effort Reduction

25%

Measures the percentage decrease in manual hours spent writing and maintaining test cases compared to the baseline before AI implementation.

Formula: ((Baseline Hours - Current Hours) / Baseline Hours) × 100
3DP Example: Before: 40 hrs/sprint writing Module 2/3 validation tests → After: 30 hrs/sprint with AI assistance = 25% reduction

Why it matters: Direct cost savings and faster delivery cycles

Coverage Increase

15%

Measures the increase in test scenarios covered, including edge cases and regulatory validation paths that were previously untested.

Formula: ((New Test Scenarios - Original Scenarios) / Original Scenarios) × 100
3DP Example: AI discovered 47 new edge cases for GxP validation and CDISC format compliance that manual testing missed

Why it matters: Reduces production defects and regulatory risks

Data Quality Score

90%

Measures the validity rate of synthetic test data generated by the AI - how often the generated data passes schema validation and business rules.

Formula: (Valid Synthetic Records / Total Generated Records) × 100
3DP Example: AI generates synthetic patient data for submission testing - 90% passes all CDISC and regulatory format checks on first attempt

Why it matters: High quality synthetic data enables testing without real patient data

Platforms Integrated

2

Count of R&D IT platforms where the Test Intelligence Agent has been deployed and is actively generating test cases.

Target: 2 primary platforms in Phase 1, extensible to additional platforms in future phases
Current: 3DP (Drug Development Data Platform) and BIKG (Biological Intelligence Knowledge Graph) — both LIVE

Why it matters: Demonstrates scalability and reusability of the agent

Breakdown: 523 regulatory validation tests, 412 data integrity tests, 189 API tests, 123 edge case scenarios

Why it matters: Shows adoption and productivity impact

Active Users

Avg Response Time

2.3s

Average time for the AI to generate a test case from a given requirement or code snippet. Lower is better.

Measured: From request submission to test case delivery (P50 latency)
Comparison: Manual test writing: ~30 minutes per test case. AI generation: 2.3 seconds = 780x faster

Why it matters: Fast responses enable real-time developer workflows

Success Rate

98.2%

Percentage of AI requests that successfully generate valid, executable test cases without errors or timeouts.

Formula: (Successful Generations / Total Requests) × 100
Failures (1.8%): Mostly due to ambiguous requirements or unsupported test frameworks - logged for improvement

Why it matters: High reliability builds user trust and adoption

1. Executive Summary

The Manufacturing Insight Agent (MIA) ontology defines a standardised knowledge graph schema that models AstraZeneca's end-to-end pharmaceutical manufacturing landscape. It captures the full lifecycle of production batches — from planned orders through process execution, quality inspection, and material genealogy — across six SAP ERP source systems spanning North America, Europe, Asia, France, Puerto Rico, and Latin America.

The ontology comprises 16 classes (node types), 23 relationship types, and 8 controlled vocabularies that together provide a unified semantic layer over three core data tables totalling 1.35 million source rows. When loaded into the Neo4j knowledge graph, the schema materialises 810,276 nodes and 2,060,803 relationships.

Purpose of this document: To present the finalised ontology schema for stakeholder review and formal sign-off, confirming that the data model accurately represents AstraZeneca's manufacturing domain and supports the intended analytical and agentic-AI use cases.

The ontology imports the AstraZeneca reference operations vocabulary (def-ops) at https://pid.astrazeneca.net/ref-ops/, ensuring alignment with enterprise-wide data standards and enabling interoperability with other knowledge graphs within the organisation.

3.5 PlannedOrder mia:PlannedOrder

Planned production order before conversion to a process order. Subclass of def-ops:Order. Linked to ProcessOrder via ORIGINATES_FROM.

3.6 InspectionLot def-ops:Lot

Quality inspection lot linked to batches. Carries inspection type (Final, In-Process, Incoming) and usage decision attributes.

3.7 LIMSLot

Laboratory Information Management System lot for lab testing, linked to batches via HAS_LIMS_LOT.

3.8 QualityEvent mia:QualityEvent

Quality events affecting batches, including deviations, out-of-specification results, and CAPAs.

3.9 QualityNotification mia:QualityNotification

Formal notifications raised for quality issues and deviations, linked to affected batches.

3.10 Plant / Site def-ops:Site

Manufacturing or planning site. Subclass of Enterprise. Linked to batches, process orders, and storage locations.

3.11 Vendor

Supplier of raw materials. Linked to batches via SUPPLIED_BY.

3.12 WorkCentre

Production resource or equipment used to execute operations.

3.13 StorageLocation

Warehouse or storage area within a plant. Classified by location type (Production, Quality Control, Warehouse).

3.14 ProductFamily

Level-1 product grouping, typically by therapeutic area (e.g., Oncology, Respiratory).

3.15 ProductSubFamily

Level-2 product grouping, typically by brand (e.g., Tagrisso, Lynparza, Symbicort).

3.16 SourceSystem

ERP source system identifier (e.g., SAP ECC EUROPE, SAP ECC NAM). Batches and orders are linked via SOURCED_FROM.

9. Business Questions This Ontology Enables

The knowledge graph enables natural-language and Cypher-based exploration of complex manufacturing questions that would be difficult or impossible to answer with relational queries alone.

9.1 Batch Genealogy Tracing

"Trace all upstream raw material batches that fed into finished product batch B-12345 at plant P100."

// Recursive batch genealogy — trace all ancestors
MATCH path = (finished:Batch {batchCode: 'B-12345', plantCode: 'P100'})
      <-[:FEEDS_INTO*1..10]-(ancestor:Batch)
MATCH (ancestor)-[:MADE_OF]->(m:Material)
OPTIONAL MATCH (ancestor)-[:SUPPLIED_BY]->(v:Vendor)
RETURN ancestor.batchCode AS batch,
       m.materialName AS material,
       v.vendorName AS supplier,
       length(path) AS depth
ORDER BY depth

9.2 Order Cycle Time Analysis

"What is the average cycle time from order creation to technical completion, by plant and product family?"

MATCH (po:ProcessOrder)-[:EXECUTED_AT]->(p:Plant),
      (po)-[:PRODUCES]->(m:Material)-[:BELONGS_TO_SUBFAMILY]->(sf)
              -[:BELONGS_TO_FAMILY]->(fam:ProductFamily)
WHERE po.actualStartDatetime IS NOT NULL
  AND po.actualFinishDate IS NOT NULL
RETURN p.plantCode AS plant,
       fam.name AS productFamily,
       avg(duration.between(po.actualStartDatetime,
           po.actualFinishDate)) AS avgCycleTime,
       count(po) AS orderCount
ORDER BY avgCycleTime DESC

9.3 Quality Event Impact Analysis

"For each quality event, find all downstream batches that may be affected through the material genealogy chain."

MATCH (qe:QualityEvent)<-[:HAS_QUALITY_EVENT]-(affected:Batch)
OPTIONAL MATCH path = (affected)-[:FEEDS_INTO*1..5]->(downstream:Batch)
MATCH (downstream)-[:MADE_OF]->(m:Material)
RETURN qe.eventId AS qualityEvent,
       affected.batchCode AS affectedBatch,
       collect(DISTINCT downstream.batchCode) AS downstreamBatches,
       collect(DISTINCT m.materialName) AS impactedMaterials

9.4 Cross-Plant Material Flow

"Show all material transfers between plants — where a batch manufactured at one plant is consumed by a process order at another."

MATCH (b:Batch)-[:MANUFACTURED_AT]->(source:Plant),
      (b)-[:CONSUMED_BY]->(po:ProcessOrder)-[:EXECUTED_AT]->(dest:Plant)
WHERE source <> dest
MATCH (b)-[:MADE_OF]->(m:Material)
RETURN source.plantCode AS fromPlant,
       dest.plantCode AS toPlant,
       m.materialName AS material,
       count(b) AS transferCount
ORDER BY transferCount DESC

10. Glossary of Key Terms

TermDefinition
OntologyA formal, shared conceptualisation of a domain, defining classes, properties, and relationships that model real-world entities and their interactions.
Knowledge GraphA graph-structured database (nodes and edges) that stores entities and their relationships, enabling traversal-based queries and pattern discovery.
Neo4jA native graph database used to store and query the MIA knowledge graph using the Cypher query language.
CypherNeo4j's declarative graph query language for pattern matching and data retrieval.
BatchA discrete quantity of material produced or procured in a single production run, uniquely identified by batch code and plant.
Process OrderAn SAP production order that defines the manufacturing steps, materials, and quantities for producing a batch.
Material GenealogyThe chain of FEEDS_INTO relationships that traces how raw material batches flow through intermediate steps to finished products.
Inspection LotA quality control unit created for testing a batch against predefined specifications.
LIMSLaboratory Information Management System — software for managing lab samples, tests, and results.
Work CentreA production resource (machine, production line, or manual workstation) where operations are executed.
def-opsAstraZeneca's reference operations vocabulary (https://pid.astrazeneca.net/ref-ops/), providing enterprise-standard class definitions.
Controlled VocabularyA predefined, finite set of allowed values for a property, ensuring data consistency across source systems.
Source SystemAn SAP ECC instance from which manufacturing data is extracted (e.g., SAP ECC NAM, SAP ECC EUROPE).
CAPACorrective and Preventive Action — a systematic process for investigating quality deviations and implementing fixes.
